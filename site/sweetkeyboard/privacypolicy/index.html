SweetKeyboard

Privacy Policy

Last updated: June 10, 2026

SweetKeyboard is built as a local-first iOS custom keyboard. The app and keyboard extension do not use analytics, advertising SDKs, tracking SDKs, cloud sync, or remote text processing.

Data Collection

SweetKeyboard does not collect personal data. It does not send keystrokes, typed text, clipboard history, settings, diagnostics, or usage events to Lafayette Consulting or to any third-party service.

Keyboard Input

Text typed with SweetKeyboard stays in the active app where you type it. SweetKeyboard does not transmit keyboard input over the network.

SweetKeyboard Clipboard

SweetKeyboard Clipboard is optional. Saved clipboard snippets and pinned favorites are stored locally on your device in the shared app container used by the SweetKeyboard app and keyboard extension.

Native iPhone Clipboard

SweetKeyboard reads native iPhone Clipboard text only after you tap a native clipboard action inside the keyboard. Clipboard text is handled as plain text and stays local on your device.

Full Access

Basic typing works without Full Access. Full Access is optional and is used only for clipboard tools and for sharing selected settings and local clipboard state between the containing app and the keyboard extension.

Network Use

SweetKeyboard does not provide a network service and does not upload typed text or clipboard content. The privacy policy page itself is a static web page with no scripts and no analytics.
